Maintain the integration master bundle in an additive, source-controlled way.

Rules:
- Keep GitHub as the source of truth.
- Preserve all existing product folders and routes.
- Add new product bundles, E2E flows, and ID-based navigations without overwriting current files.
- Update dashboard, manifest, README, and launcher pages only when needed.
- Generate CSV, XLSX, JSON, TS, and HTML outputs in each module folder.
- Package the final artifact as a zip and open a PR.
- Ensure mobile-friendly navigation, PWA install support, feedback form routing, and About page coverage remain intact.
